# 华润电力文档格式规范

> 适用范围：运行日志、事故/异常报告、技术方案、培训总结、操作票/工作票
> 版本：v1.0 | 最后更新：2026年7月

---

## 一、通用规范

所有文档统一遵守以下基础规范：

### 1.1 页面设置

| 项目 | 标准值 |
|------|--------|
| 纸张 | A4（210mm × 297mm） |
| 上边距 | 2.54cm |
| 下边距 | 2.54cm |
| 左边距 | 3.17cm |
| 右边距 | 3.17cm |

### 1.2 字体与字号

| 层级 | 字体 | 字号 | 粗细 | 备注 |
|------|------|------|------|------|
| 文档标题 | 微软雅黑 / 黑体 | 小二号（18pt） | 加粗 | 居中 |
| 一级标题 | 黑体 | 三号（16pt） | 加粗 | 左对齐 |
| 二级标题 | 黑体 | 小三号（15pt） | 加粗 | 左对齐 |
| 三级标题 | 黑体 | 四号（14pt） | 加粗 | 左对齐 |
| 正文 | 宋体 | 小四号（12pt） | 常规 | 两端对齐 |
| 页眉 | 宋体 | 小五号（9pt） | 常规 | 顶部居中 |
| 页码 | 宋体 | 小五号（9pt） | 常规 | 底部居中 |

### 1.3 行距与段落

| 项目 | 标准 |
|------|------|
| 正文行距 | 1.5倍行距 |
| 标题行距 | 单倍行距 |
| 段落间距 | 段前0行，段后0.5行 |
| 首行缩进 | 2个字符（正文段落） |

### 1.4 页眉与页脚

- **页眉内容**：`华润电力 · [文档类型名称]`
- **页眉位置**：顶部居中
- **页码格式**：`第 X 页 / 共 Y 页`
- **页码位置**：底部居中

## 二、各文档类型特殊规范

### 2.1 运行日志

**适用场景**：日常班次运行记录

**格式要求**：
- 以表格为主体
- 表头包含：日期、班次（白班/中班/夜班）、机组编号、主要参数
- 异常记录单独标注（加粗或红色标记）
- 时间格式统一为 `YYYY-MM-DD HH:mm`（24小时制）
- 底部签名栏：
  ```
  记录人：______    审核人：______
  日期：YYYY年MM月DD日
  ```

**结构模板**：
```
[文档标题：XX机组运行日志]

日期：YYYY-MM-DD    班次：白/中/夜    值班人：XXX

┌──────┬──────┬──────┬──────┬──────┐
│ 时间 │ 参数 │ 数值 │ 状态 │ 备注 │
├──────┼──────┼──────┼──────┼──────┤
│      │      │      │      │      │
└──────┴──────┴──────┴──────┴──────┘

异常记录：
[如有异常，在此描述]

交接班说明：
[交接内容]

签名：______
```

### 2.2 事故/异常报告

**适用场景**：设备故障、安全事件、运行异常等

**文档结构**（按顺序）：

| 章节 | 内容要求 |
|------|---------|
| 标题 | `关于[日期][机组/设备][事件类型]的异常报告` |
| 一、事件经过 | 时间、地点、现象、发现人，按时间线描述 |
| 二、原因分析 | 直接原因 + 间接原因，可使用鱼骨图附在附录 |
| 三、处理措施 | 已采取的措施及结果 |
| 四、经验总结 | 从事件中学到了什么，可复用的经验 |
| 五、后续计划 | 整改措施、责任部门、完成时限 |
| 落款 | 部门 + 日期 + 负责人签名 |

**标题示例**：
```
关于2026年7月8日#2机组汽轮机振动异常的异常报告
```

**紧急程度标注**：在标题右上角标注「一般 / 重要 / 紧急」

---

### 2.3 技术方案

**适用场景**：技术改造、设备升级、项目实施

**文档结构**：

| 章节 | 内容要求 |
|------|---------|
| 一、背景与目的 | 为什么做、要达到什么效果 |
| 二、现状分析 | 当前情况、存在问题、数据支撑 |
| 三、技术路线 | 技术方案对比、选型依据、关键技术指标 |
| 四、实施方案 | 步骤、进度安排、资源配置 |
| 五、预期效果 | 量化指标（效率提升/成本降低/安全改善） |
| 六、预算估算 | 费用明细及资金来源 |
| 七、风险评估 | 潜在风险及应对措施 |

**图表要求**：
- 所有图片标注：`图1 + 空格 + 图名`（居中置于图片下方）
- 所有表格标注：`表1 + 空格 + 表名`（居中置于表格上方）
- 正文中应有引用，如"如图1所示"、"详见表1"

**参考文献格式**：
```
[1] 作者. 题名[文献类型标识]. 期刊名, 年, 卷(期): 起止页码.
[2] 作者. 书名[M]. 出版地: 出版社, 出版年.
```

---

### 2.4 培训总结/心得体会

**适用场景**：培训结业、学习总结、轮岗心得

**文档结构**：

| 章节 | 内容要求 |
|------|---------|
| 一、培训概况 | 培训时间、地点、内容、讲师 |
| 二、主要内容 | 培训的核心知识点回顾 |
| 三、个人收获 | 结合岗位的真实体会（最核心部分） |
| 四、改进建议 | 对培训或工作的建设性意见 |
| 五、下一步计划 | 如何将所学应用到实际工作中 |

**字数建议**：800-1500字
**语气要求**：第一人称，真实诚恳，避免空话套话

---

### 2.5 操作票 / 工作票

**适用场景**：设备操作、检修作业

**格式要求**：
- 严格按表格模板填写，逐项勾选确认
- 不得涂改，填写错误处需签章更正
- 操作顺序不可颠倒
- 签名栏必须包含：操作人、监护人、值班负责人

**示例签名栏**：
```
操作人：______    监护人：______
值班负责人：______    日期：YYYY年MM月DD日
```

## 三、常见格式错误清单

| 序号 | 常见错误 | 正确做法 |
|------|---------|---------|
| 1 | 标题层级混乱（如跳级） | 一级→二级→三级，不可从一级直接跳到三级 |
| 2 | 正文忘记首行缩进 | 每段首行缩进2字符 |
| 3 | 图表无编号无标题 | 所有图表必须编号且标注标题 |
| 4 | 落款位置不统一 | 统一放在文档末尾右下角 |
| 5 | 时间格式不统一 | 全文使用同一格式 `YYYY-MM-DD HH:mm` |
| 6 | 行距不一致 | 正文统一1.5倍行距 |

## 四、文件命名规范

```
[文档类型]_[日期]_[主题]_[版本号].扩展名

示例：
异常报告_20260708_2号机组振动异常_v1.0.docx
运行日志_20260709_白班_3号机组.xlsx
技术方案_202607_输煤系统改造_v2.1.docx
培训总结_2026新员工训练营_王某某.docx
```
